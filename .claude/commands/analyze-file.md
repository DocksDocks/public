# Analyze File: $ARGUMENTS

Perform a comprehensive analysis of the specified file.

## Analysis Checklist

### Code Quality
- Readability and clarity
- Function/method complexity
- Naming conventions
- Comments and documentation
- Code organization

### Potential Issues
- Bugs and logic errors
- Security vulnerabilities
- Performance problems
- Error handling gaps
- Edge cases not covered

### Improvement Suggestions
- Refactoring opportunities
- Better patterns to use
- Missing tests
- Documentation needs

## Output
Provide a detailed report with:
1. **Summary**: What this file does
2. **Issues Found**: Prioritized list
3. **Recommendations**: Specific improvements
4. **Code Examples**: Before/after for suggested changes
